01 — Objective

Project Objective

The goal of this lab is to simulate the OSINT reconnaissance phase of an authorized penetration test or threat intelligence assessment. Using SpiderFoot's automated scanning engine, I collected and correlated publicly available data about a simulated target organization, identifying exposed subdomains, associated email addresses, leaked credentials references, IP intelligence, and potential attack vectors — without interacting directly with the target's systems.

This project demonstrates competency in passive reconnaissance techniques applicable to red team engagements, vulnerability assessments, and defensive exposure audits.

03 — Methodology

Scanning Methodology

The engagement followed a structured passive reconnaissance workflow, divided into four phases aligned with industry-standard OSINT methodology:

Phase Activity Data Collected SpiderFoot Mode
01 Target Seeding Domain name, org name, ASN, IP ranges Passive
02 Footprinting Subdomains, DNS records, WHOIS, SSL certs, email addresses Footprint
03 Threat Intel Correlation Blacklist presence, breach data, malicious IP flags, dark web mentions Investigate
04 Analysis & Reporting Graph visualization, correlation analysis, risk classification All Output Formats

The Passive mode was used first to build a broad intelligence baseline without any direct contact with target systems, ensuring complete non-intrusiveness. The Footprint mode was then applied to surface deeper technical infrastructure details.

07 — Recommendations

Security Recommendations

Based on the intelligence gathered, the following remediation actions are recommended, ordered by priority:

  • R-01
    Immediate Credential Reset — All email addresses identified in breach databases should have passwords reset immediately and MFA enforced. Cross-reference with internal SSO and VPN access logs for potential unauthorized access.
  • R-02
    Restrict Exposed Admin Panel — The admin subdomain discovered via CT logs should be moved behind a VPN gateway or IP allowlist. Exposed admin panels are primary targets for brute-force and credential stuffing attacks.
  • R-03
    Certificate Lifecycle Management — Implement automated certificate monitoring (e.g., via Let's Encrypt with auto-renewal or Certbot) to prevent unplanned SSL expiry. Expiry on production services can cause service disruption and loss of user trust.
  • R-04
    GitHub Repository Audit — Conduct an organization-wide audit of all public and private employee GitHub repositories for hardcoded credentials, internal URLs, API keys, and configuration files. Implement GitHub secret scanning alerts.
  • R-05
    Attack Surface Review — Inventory all 27 discovered subdomains and decommission or restrict those that are unused (dev, staging, legacy). Consider implementing DNS monitoring to detect unauthorized subdomain registration (subdomain takeover risk).
  • R-06
    HTTP Header Hardening — Remove or sanitize Server, X-Powered-By, and framework version headers from all public-facing web servers to limit technology fingerprinting by adversaries.
